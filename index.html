Listado de los comando de git y sus descripciones

Primeros Comandos en Git

  • git init: Activa git en nuestro proyecto creando un espacio en RAM llamado "STAGING" y crea una carpeta .git
  • git add: este comando deja en staging o en estado de espera los cambios del elemento mencionado o de todos los elementos de nuestro directorio en memoria ram para luegos hacerles un commity así guardar los cambios en una nueva version del proyecto.
  • git commit: envía los archivos con sus cambios al repositorio
  • git status te dice el estado de tu repositorio, te advierte de guardar cambios de tus nuevos archivos y también te avisa si se realizaron cambios en alguno de ellos para que los guardes y no pierdas tus procesos. :)

Primeros Comandos en Git

  • git log: muestra el historial de versiones o commits que se realizaron en el archivo.
  • git show: muestra que cambios se realizaron en cada una de las versiones.
  • git diff: compara entre las versiones del archivo.
  • merge: se utiliza para unir una rama alterna a la rama Máster
  • checkout: sirve para traer un cambio desde el repositorio al directorio.
