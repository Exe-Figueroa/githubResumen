Listado de los comando de git y sus descripciones

Primeros Comandos en Git

  • git init: Activa git en nuestro proyecto creando un espacio en RAM llamado "STAGING" y crea una carpeta .git
  • git add: este comando deja en staging o en estado de espera los cambios del elemento mencionado o de todos los elementos de nuestro directorio en memoria ram para luegos hacerles un commity así guardar los cambios en una nueva version del proyecto.
  • git commit: envía los archivos con sus cambios al repositorio
  • git status te dice el estado de tu repositorio, te advierte de guardar cambios de tus nuevos archivos y también te avisa si se realizaron cambios en alguno de ellos para que los guardes y no pierdas tus procesos. :)

Navegando entre versiones

  • git log: muestra el historial de versiones o commits que se realizaron en el archivo.
  • git show: muestra que cambios se realizaron en cada una de las versiones.
  • git diff: compara entre las versiones del archivo.
  • merge: se utiliza para unir una rama alterna a la rama Máster
  • checkout: sirve para traer un cambio desde el repositorio al directorio.

Comando reset y remove

  • GIT RESET: es un comando peligroso que elimina todo sin posibilidad de volver a recuperar lo perdido y debe ser utilizado en caso de emergencia unicamente.
  • git reset --hard: Borra absolutamente todo, tanto en el directorio como en el repositorio.
  • git reset --soft: boora todo el historial y registro de git pero conserva los cambios del staging.
  • git reset HEAD: elimina los cambios del área de staging para que estos no se guarden con el commit.
  • git rm --cached: elimina los cambios en staging pero los mantiene en el disco duro.
  • git rm --force: elimina los cambios en git y en directorio

Flujo de trabajo con un repo remoto

  • git clone +url: trae una copia del master a mi directorio local y crea una base de datos de los cambios en el repo local
  • git push: envia los cambios al repo remoto
  • git fech: trae los cambios realizados desde el repo remoto al local
  • git merge: fusiona los cambios
  • git pull: es la combinación de los dos conceptos anteriores

Intro a los Branchs de GIT

  • git commit -am +comentario: hace un add y un commit de los archivos (con un git add . ejecutado anteriormente).
  • git branch +name: crea una rama con el nombre asignado.
  • git checkout +branch: mueve el head desde el máster a la rama asignada.
  • git checkout -b +branch: combina los dos conceptos anteriores
  • git checkout +branch -o-id -códigoDelCommit: te mueve al commit especificado sin borrar los datos posteriores del tag seleccionado.

Fusión de branchs con git merge

  1. Colocarse en el master: para que la rama secundaria no se vuelva la principal.
  2. Ejecutar comadno: git merge +branch +"comentario"
  3. Ejecutargit branch en caso de no conocer los nombres de las ramas.

Creando un repo remoto en GitHub

  1. Colocar nombre del repo.
  2. Colocarle una descripción.
  3. Inicializarlo con un Readme.md ya que esta es una buena práctica y permite redactar un resumen de tu proyecto.
  4. Declarar si es público o privado.
  5. En clone copiar url. ssh o html
  6. En la terminal de nuestro repo local agregar repo remoto.
  7. Usar Comando: git remote add origin +url.
  8. Realizar git pull origin master para trarer todo lo del repo remoto al local.
  9. En caso de que el merge no se produzca colocar: git pull origin master --allow-unrelated-histories para forzar la fusión.

Tags y versiones en git y GitHub

  • git log: muestra una lista de los commits realizados en el repo
  • git log --all --graph --decorate --oneline: arma un arbolito detallado y bien bonito con los commits realizados.
  • Alias de un comando: alias + nombre de alias + ="comando"

    • Crear un tag

  • Copiar el código del commit
  • Ejecutar: git tag -a + versión + -m + "nombre del commit"
  • git tag: muestra los tags
  • git tag -d + nombre: elimina el tag
  • git push origin :refs/tags/nombre del tag: lo borra del repo de git

Manejo de ramas en GitHub

  • git show-branch: muestra las ramas
  • git show-branch --all: lo mismo pero más detalladas
  • gitk: abre el arbolito en una interfaz visual
  • git push + nombre del branch: envía determinada rama al repo en GitHub
